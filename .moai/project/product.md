# Korea Real Estate MCP - Product Definition

## 프로젝트 이름

**Korea Real Estate MCP** (한국 부동산 MCP)

## 설명

Model Context Protocol (MCP) 기반의 한국 부동산 데이터 조회 서버입니다. Claude Desktop 및 웹 클라이언트에서 한국 아파트 가격 정보를 자연어로 조회할 수 있게 해주는 17개의 MCP 도구를 제공합니다.

공공데이터포털의 오픈 API를 활용하여 부동산 매매가, 전월세가, 청약 정보, 온비드 공매 정보 등을 신뢰성 높게 제공합니다.

## 타겟 사용자

### 1차 사용자
- **Claude Desktop 사용자**: 한국 부동산 시장 데이터가 필요한 일반 사용자
- **부동산 투자자**: 지역별 아파트 가격 추이를 분석하고자 하는 투자자
- **부동산 중개사**: 고객에게 정확한 시세 정보를 제공해야 하는 중개사

### 2차 사용자
- **데이터 분석가**: 부동산 시장 동향을 분석하는 연구원
- **개발자**: 부동산 API를 활용하여 서비스를 개발하는 개발자

## 핵심 기능

### 1. 부동산 시세 조회
- **아파트 매매가**: 지역별 아파트 매매 실거래가 조회
- **아파트 전월세가**: 지역별 아파트 전월세 실거래가 조회
- **다가구/다세대 매매가**: 다가구/다세대 주택 매매 실거래가 조회
- **다가구/다세대 전월세가**: 다가구/다세대 주택 전월세 실거래가 조회
- **연립/빌라 매매가**: 연립/빌라 주택 매매 실거래가 조회
- **연립/빌라 전월세가**: 연립/빌라 주택 전월세 실거래가 조회
- **단독/다가구 매매가**: 단독/다가구 주택 매매 실거래가 조회
- **단독/다가구 전월세가**: 단독/다가구 주택 전월세 실거래가 조회
- **토지 매매가**: 토지 매매 실거래가 조회
- **분양권 매매가**: 분양권 매매 실거래가 조회

### 2. 청약 정보
- **청약 당첨률**: 아파트 청약 당첨률 조회
- **청약 커트라인**: 아파트 청약 경쟁률 및 커트라인 조회

### 3. 온비드 공매
- **공매 물건 조회**: 법원 경매, 공매 물건 정보 조회
- **입찰 정보**: 공매 입찰 진행 현황 및 결과 조회

### 4. 기술적 기능
- **OAuth 2.0 인증**: Client Credentials 및 Auth0 JWT 방식 지원
- **API 응답 캐싱**: TTL 5분 기반 캐싱으로 API 호출 최적화
- **네트워크 신뢰성**: 재시도 메커니즘 및 Circuit Breaker 패턴
- **입력 검증**: 지역코드, 날짜 형식 사전 검증
- **표준 오류 처리**: 6가지 오류 타입 기반 안정적인 오류 처리

## 유스케이스

### UC-1: 지역별 아파트 시세 조회
**사용자**: 서울 강남구 아파트 가격을 알고 싶은 일반 사용자

**시나리오**:
1. 사용자가 Claude에 "서울 강남구 삼성동 아파트 최근 매매가 알려줘"라고 질문
2. Claude가 `get_apt_trade_prices` 도구를 호출하여 지역코드(11680)와 기간 설정
3. MCP 서버가 공공데이터포털 API에 요청
4. XML 응답을 파싱하여 구조화된 데이터 반환
5. Claude가 사용자에게 이해하기 쉽게 결과 요약 전달

### UC-2: 전월세 시장 동향 분석
**사용자**: 부동산 투자자

**시나리오**:
1. 사용자가 "최근 3개월간 서울 아파트 전세가 동향 분석해줘"라고 요청
2. Claude가 `get_apt_rent_prices` 도구를 월별로 3회 호출
3. 각 월별 데이터를 취합하여 동향 분석
4. 월별 가격 변화율 계산 및 시장 트렌드 도출
5. 사용자에게 인사이트 제공

### UC-3: 청약 당첨 가능성 확인
**사용자**: 아파트 청약을 준비하는 예비 청약자

**시나리오**:
1. 사용자가 "이번달 서울 강남구 청약 당첨률 어때?"라고 질문
2. Claude가 `get_subscription_acceptance_rate` 도구 호출
3. 해당 지역의 최근 청약 당첨률 데이터 반환
4. 평균 경쟁률 및 당첨 확률 분석
5. 사용자에게 청약 전략 제안

### UC-4: 공매 물건 검색
**사용자**: 저렴한 부동산을 찾는 투자자

**시나리오**:
1. 사용자가 "서울 지역 5억 이하 공매 물건 찾아줘"라고 요청
2. Claude가 `get_onbid_auction_goods` 도구 호출
3. 온비드 API에서 조건에 맞는 공매 물건 검색
4. 매각 예정가, 위치, 물건 상태 등 상세 정보 제공
5. 사용자가 입찰 참여 여부 결정 지원

### UC-5: 대규모 데이터 요청 처리
**사용자**: 데이터 분석가

**시나리오**:
1. 사용자가 "2024년 전국 아파트 매매가 데이터 모아서 CSV로 만들어줘"라고 요청
2. Claude가 지역별로 `get_apt_trade_prices` 도구 순차 호출
3. 캐싱 메커니즘으로 중복 API 호출 방지
4. 수집된 데이터를 구조화하여 CSV 형식 변환
5. 사용자에게 파일 제공

## 기대 효과

### 사용자 효용
- 복잡한 공공데이터포털 API 직접 사용 없이 자연어로 데이터 조회
- 정확하고 신뢰할 수 있는 공공 데이터 제공
- 다양한 부동산 데이터 통합 조회 가능

### 기술적 우수성
- MCP 표준 준수로 Claude Desktop 및 웹 클라이언트 호환
- 안정적인 API 연동을 위한 재시도 및 캐싱 메커니즘
- 안전한 XML 파싱으로 보안 위험 방지
- 포괄적인 테스트 (370개 테스트, 87.33% 커버리지)
